FunYums!

OUR GOAL

FunYums! strives to offer cooking users of all experience levels food recipes and suggestions based on the ingredients a user already owns. The ability for users to utilize information like expiration date and ingredients will help people save money and reduce waste by efficiently using their ingredients. We also hope to motivate individuals to cook more with our points system. Have more fun, produce less waste, and save more money, one recipe at a time!

DATA AND TOOLS

For our tools, we will be using Angular for our front-end. This will allow us to create our front-end webpages in a simple and intuitive manner using different components. For our application, we will create different components for each page, for example, one for a recipe page, one for our about page, etc. For the deployment of our application, we will be using Google Cloud Platform, which will allow us to easily move our app to live production that can be viewed by anyone on the internet. Another reason for using this is that the majority of the team has experience with launching apps to GCP. For testing, we used Karma with Jasmine. This allowed us to quickly deploy JavaScript tests in a real browser which includes a native debugger.

The primary data source we are using is the Yummly API: https://developer.yummly.com/documentation.
This API provides recipe search features allowing us to obtain ingredients, pictures, step-by-step instructions, and nutritional facts for a given recipe by using HTTP requests and parsing the resulting JSON data.
